NOTE: This document was written when Docker was the planned container runtime.
The project has since migrated to Podman. All Docker/docker-compose references
in this document reflect the original plan.

1. Project Overview

Build a production-grade Django web application using:

Python 3.12+

Django (latest LTS)

PostgreSQL

Docker + Docker Compose

CI/CD via GitHub Actions

Full test coverage

Secure authentication system

Environment-based configuration

Production deployment readiness

The application must follow clean architectural separation without unnecessary enterprise-level abstraction.

2. Architectural Principles
Required Design Approach

Feature-based Django apps (modular structure)

Separation of concerns:

Models (data)

Services (business logic)

Views / API layer

Serializers (if using DRF)

Business logic must NOT live inside views

No “fat models” — use service layer for complex logic

Configuration must be environment-driven

All secrets must be environment variables
